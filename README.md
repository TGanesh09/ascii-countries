I created these as part of building [Where In the World](https://github.com/acidus99/WhereInTheWorld), a text-based [Worldle](https://worldle.teuteuf.fr) clone for [Gemini](https://gemini.circumlunar.space/).

I tried a lot of different ASCII art converters,  but ultimately went with [ascii-image-converter's braille character option](https://github.com/TheZoraiz/ascii-image-converter). So, technically these aren't ASCII art, but Unicode art. The extra dots of the braille characters allowed me to have finer detail for the borders, while still keeping the images small.

## Build script
These files are generated by a build script `country-convert.sh`. As input, it takes [the directory of SVG images](https://github.com/teuteuf/worldle/tree/main/public/images/countries) of territories collected by the [Worldle repository](https://github.com/teuteuf/worldle/).

* Renames SVG images with the ISO-3166 two letter code
* Converts them to 1024 x 1024 PNGs
* Converts them to braille ASCII art `.txt` files at the appropriate width
* Moves them to `output/` directory
